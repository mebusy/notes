# [tinyrenderer](https://github.com/ssloy/tinyrenderer)

tinyrenderer（常指 Dmitry V. Sokolov 的 Tiny Renderer 教程/小型软渲染管线实现）的核心优势不在“工业级可用”，而在于用极少的代码把现代渲染管线的关键概念端到端跑通。因此它最适合的应用场景是“学习、验证、原型、研究”，而不是直接拿来做商业渲染器。

优势应用场景（最匹配的几类）

1) 图形学/渲染管线学习与教学
    - 从顶点处理 → 光栅化 → 插值 → 深度测试 → 着色（在 CPU 上）把流程讲清楚
    - 适合课堂作业、读书会、内部培训
    - 优势点：代码量小、依赖少、便于逐行调试，理解“为什么这么做”比“API 怎么用”更直接

2) 快速验证渲染算法与数学正确性（CPU 参考实现）
    - 在上 GPU 之前，用 CPU 版先把：
    - barycentric 插值是否正确
    - 深度/透视矫正是否正确
    - 法线/切线空间变换是否正确
    - 简化版 BRDF/光照模型是否合理
    - 做一个可控的 reference
    - 优势点：可复现、易断点、易打印中间结果，适合做“对照组”

3) 研究/论文原型：新着色模型或可见性方法的小规模实验
    - 你想测试一个想法（比如一种新的采样策略、简化 GI、特殊材质模型），不想先把 Vulkan/DX12/驱动细节搭起来
    - 优势点：把注意力集中在算法本身；渲染管线可按需要改造
4) 调试与排错：理解常见渲染 Bug 的来源
    - 特别适合用来“构造最小复现”并理解：
    - Z-fighting、深度精度问题
    - 背面剔除、绕序（winding order）错误
    - 透视矫正插值 vs 线性插值导致的纹理扭曲
    - 法线插值/空间变换错误导致的高光异常
    - 优势点：可以把每一步输出成图片或数值，定位非常快

5) 作为 GPU 渲染器的“金标准/回归测试”基线（小规模）
    - 在引擎里做功能时，常需要一个“独立实现”来做对比（尤其是几何处理、插值、深度逻辑）
    - 优势点：无需 GPU 即可跑在 CI 上做基本正确性回归（注意：性能和完整性不适合作为全量测试）

6) 嵌入式/无 GPU 环境的极简渲染需求（非常有限）
    - 在没有 GPU 或图形 API 的环境里做简单离线输出（比如生成缩略图、教育演示）
    - 但要强调：tinyrenderer 更偏“教学性质”，若要产品化通常会选择更成熟的软渲染方案或专用库

不太适合的场景（它的“优势”反过来的地方）
    - 实时高性能渲染（CPU 光栅化很难和 GPU 比）
    - 复杂现代图形特性全家桶：PBR 全套、阴影系统、后处理链路、粒子、GPU 驱动优化等
    - 作为通用游戏引擎渲染后端（维护成本和性能都不划算）
